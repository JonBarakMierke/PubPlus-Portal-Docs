Reusable, themeable call-to-action section. Renders as `<x-sections.cta>`.

## File Structure

```text
resources/views/components/sections/cta/
├── index.blade.php          # Router — resolves variant, theme, and shared props
└── variants/
    ├── centered.blade.php   # Centered, single-column layout
    └── split.blade.php      # Two-column layout with an illustration/image
```

> ⚠️ Never let a flat `sections/cta.blade.php` file exist alongside this directory. Laravel resolves the flat file first and will silently ignore everything below.

---

## Props

| Prop | Type / Values | Default | Notes |
| :-- | :-- | :-- | :-- |
| `variant` | `centered` \| `split` | `centered` | Picks which file in `variants/` renders. |
| `theme` | `dark` \| `light` \| `brand` \| `navy` | `dark` | Controls background \+ all text colors as a matched set. See [Themes](https://claude.ai/chat/8100338c-0503-43da-92d6-0b3e432b87f4#themes). |
| `container` | any Tailwind `max-w-*` class | `max-w-4xl` | Controls the max width of the section content. |
| `rounded` | any Tailwind rounding class | `rounded-4xl` | Outer container corner radius. |
| `imagePosition` | `left` \| `right` | `right` | `split `**variant only.** Which side the illustration sits on. |
| `imageWidth` | any Tailwind width class | `md:w-1/2` | `split `**variant only.** Width of the image column at `md` breakpoint. |
| `imageStyle` | `contained` \| `bleed` | `contained` | `split `**variant only.** See [Image Styles](https://claude.ai/chat/8100338c-0503-43da-92d6-0b3e432b87f4#image-styles). |

Kebab-case in Blade tags, e.g. `image-position="left"`, `image-style="bleed"`.

---

## Slots

| Slot | Required | Description |
| :-- | :-- | :-- |
| `eyebrow` | No | Small uppercase label/pill above the heading. |
| `heading` | No | Main `<h2>` (rendered via `x-ui.heading`). |
| `subtext` | No | Supporting paragraph under the heading. |
| `actions` | No | Button(s) — typically an `x-ui.button`. |
| `footer` | No | Secondary links, e.g. contact info. |
| `illustration` | `split` only | Image, SVG, or other visual for the image column. Ignored by `centered`. |

All slots are optional and independently `@isset`-guarded — omit anything you don't need.

---

## Themes

Themes are a fixed, brand-approved set defined once in `index.blade.php`. There is **no free-form** `bg `**prop** — this is intentional, so every CTA on the site stays on-brand.

| Theme | Background | Text | Use case |
| :-- | :-- | :-- | :-- |
| `dark` | Neutral gradient (`neutral-400/50 → neutral-500/80`) | White / light gray | Default, general-purpose CTA. |
| `light` | `bg-sky-50` | Dark neutral tones | Soft, editorial sections (e.g. "Support at Every Step"). |
| `brand` | `bg-grad-accent-left` (site accent gradient) | White | High-emphasis, promotional CTAs. |
| `navy` | `bg-neutral-900` | White / light gray | Placeholder — update with real brand navy if/when defined. |

**To add a new theme:** add one entry to the `$themes` array in `index.blade.php` with `bg`, `heading`, `subtext`, `footer`, and `eyebrow` keys. It becomes available everywhere immediately — no changes needed in variant files.

---

## Image Styles (`split` variant)

- `contained` (default) — Illustration floats freely within its column (e.g. a rotated product mockup with its own shadow/rounding). Use for SVGs, product shots, or anything that should look "placed," not full-bleed.
- `bleed` — Illustration fills its entire column edge-to-edge, including the section's outer padding, and takes on the section's outer rounding on its side. Use for photography meant to fill the space (e.g. team photos, lifestyle shots). Pass an `<img class="w-full h-full object-cover ...">` — `object-position` can be adjusted per image (e.g. `object-left`, `object-top`) to control the crop.

## Adding a New Variant

1. Create `resources/views/components/sections/cta/variants/{name}.blade.php`.
2. Add an entry to the `$variants` array in `index.blade.php`: 
   ```php
   $variants = [    'centered' => 'components.sections.cta.variants.centered',    'split'    => 'components.sections.cta.variants.split',    'new-name' => 'components.sections.cta.variants.new-name',];
   ```
3. The new file automatically receives every prop and slot (`$theme`, `$palette`, `$bg`, `$container`, `$rounded`, `$eyebrow`, `$heading`, `$subtext`, `$actions`, `$footer`, `$illustration`, etc.) via the shared `@include($view)` — no extra wiring needed.
4. Reference `$palette['heading']`, `$palette['subtext']`, `$palette['footer']`, and `$palette['eyebrow']` for text colors so the new variant respects themes automatically, rather than hardcoding colors.

## Known Gotchas

- **Flat file collision:** `sections/cta.blade.php` (file) takes priority over `sections/cta/index.blade.php` (directory) if both exist. Always delete the flat file when migrating a component to the directory pattern.
- **Stale compiled views:** run `php artisan view:clear` after moving/renaming component files — Blade's compiled cache can mask changes mid-development.
- `bleed `**image style** uses negative margins to escape the section's padding. If you change the section's padding values in `split.blade.php`, double-check the bleed offsets (`-mx-8 md:-mx-14` etc.) still align.
- **No free-form colors:** intentionally, there's no `bg` prop exposed on `<x-sections.cta>`. New color needs should go through adding/editing a theme in `index.blade.php`, not a one-off class on a page.
